2.2 Developer Use Cases

2.2.1 Use Case 1: Enable Caching

Properties
Primary Actor: Software Developer
Level: User-goal
Success Guarantee: Caching is enabled
Minimal Guarantee: ---
Preconditions: 1. System is accessible
2. Caching is disabled
Main Success Scenario
1. The Primary Actor indicates to the System that they wish to activate caching
2. The System provides a form for the Primary Actor to set the amount of disk space and/or memory to be dedicated to caching
3. The Primary Actor enters the appropriate data and submits
4. The System confirms that caching has been enabled
5. Use Case ends successfully
Extensions
3.a. The data provided by the Primary Actor is wrong
3.a.1. The system returns an error message to the Primary Actor specifying why the data is not acceptable (if applicable)
3.a.2. Use case resumes at Step 3

2.3.4 Use Case 14: Install/Setup Boogie

Properties
Primary Actor: Admin
Level: User-goal
Success Guarantee: The system will now be able to use Boogie for verification.
Minimal Guarantee: ---
Preconditions
1. System is accessible
2. Boogie is installed on the system.
Main Success Scenario
1. The Primary Actor specifies that they want to configure settings for JML4 Disco.
2. The Primary Actor specifies that they want to install/setup Boogie.
3. The System prompts user for the path to Boogie.
4. The Primary Actor specifies the path to Boogie.
5. The System acknowledges the path and presents the system as being setup to use Boogie.
6. Use Case ends successfully
Extensions
5.a. Path Specified is invalid.
5.a.1. The system returns a warning indicating that the path specified by the user is invalid.
5.a.1.a The user corrects the specified path to Boogie.
5.a.1.a.1 The system goes to step 5.
5.a.1.b The user cancels the setup.
5.a.1.b.1 System does not setup Boogie.
