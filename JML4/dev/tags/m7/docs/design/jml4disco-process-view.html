2.2 Relevant Components

* ProveVcProgram - Is the initial servlet that processes the clients request. It is responsible for sending the VC program to the Dispatcher.
* VcProgramDispatchingServer - Is the dispatcher and will "break" the VCprogram into VCs. For each Vc, as suitable server is found and a ProveVcThread is made and run by the process pool.
Please note: a throttle mechanism has been added to "pause" dispatching temporarily if the most suitable server is overburdened with pending VCs.
* ProveVcThread - Actually just implements Runnable. It calls the proveVc(...) method from the passed server parameter and accumulates the result.
* ProveVc - This is the servlet that must be deployed on "servant" servers (those that do not dispatch).
* ProveVcServer - This contains only a static method prove(...) which calls the ProveVcPiecewise from original ESCjava4.

3. Boogie Components


The Process view of the Boogie Component of the application is quite simple as the application uses one thread for the Boogie Strategy . When the application is running and the Boogie strategy is enabled, the preCodeGeneration will call the Boogie process method which in turn will run one process of the BoogieAdapter. The BoogieAdapter is responsible for taking generated Boogie source from the BoogieVisitor (AST Visitor), writing it to a source file, and running it through Boogie (the executable).
