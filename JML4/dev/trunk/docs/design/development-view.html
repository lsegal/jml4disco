Milestone 4

The Distributed Multi-Threaded Program Verifier component of JML4Disco is split into various layers, as described in the accompanying architecture document covering the Logical View. In terms of development, each layer is its own Eclipse Java Project, as follows:

  • The topmost layer (the presentation layer) which is responsible for communication via Http protocol is currently contained in the project org.jmlspecs.jml4.esc.distributedServlets. It contains only the implementation of Servlets, and should be renamed to org.jmlspecs.jml4.esc.distribution.http.

  • The application layer controls all logic relevant to the task of distributing program verification, and utilizing resources on remote machines. The classes of the application layer are found in the project org.jmlspecs.jml4.esc.distribution.

  • The layers below are relevant to the logic on verification, and consist essentially of all other projects which were contained in JML4 before the beginning of the JML4Disco project.

Milestone 5

Development will begin this milestone on web-based configuration interfaces for JML4Disco servers used in distributed program verification. The intent is for the creation of new Eclipse Projects to represent separate layers of the two new components: (one for each type of server, Dispatcher and Prover)

  • Two web project which will contain Servlets meant to serve as a presentation layer, bridging the gap between Http requests/responses and the underlying interface-independent configuration system.
  • An application layer project, which will contain implementations of Front Controllers and Commands being executed. So as to be able to reuse Front Controller and Command implementations, this project should be shared and contain Commands meant to configure either a Dispatcher or Prover machine, and the presentation layer should be responsible for ensuring that irrelevant Commands are inaccessible. (for example, on a Dispatcher there would be no need to configure settings for logical provers, and likewise on a Prover Server there would be no need to allow access to Commands to add or remove remote servers with which to communicate)
    • It may be beneficial to use an existing web library (such as SoenEA) instead of developing the Front Controller and Command classes from scratch.
  • The domain logic layer beneath these would consist of the existing JML4Disco projects, in particular the org.jmlspecs.jml4.esc.distribution which would contain most of the configurable settings.
