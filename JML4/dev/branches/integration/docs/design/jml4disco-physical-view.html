Physical View (Deployment View)

This section will describe the physical (deployment) view of the JML4Disco distributed subsystem. Each Node represents a type of physical computer / terminal and each square inside it represents an important element that originates from the entity that surrounds it. The entities are connected by a network protocols (e.g. HTTP). Milestone 4

The deployment view has slightly changes to reflect the design changes to improve system performance. The overall communication network did not change. All client/server communications are still done through HTTP. Nevertheless, the VC dispatcher is now running in client machine directly rather than having a dedicated server to receive all clients' request (VcProgram) then further dispatch VCs to servers for proving. This change decision was made to avoid creating another performance bottleneck on the dispatcher server. For instance, if there's 1000 clients requesting for proving their programs, there will be congestion on the dispatcher server which will be overloaded on processing requests as well as dispatching VCs to other servers for proving. In addition, each VC proving servers are now spawning only one single process for Simplify and Isabelle theorem provers. This changes improve significantly the system performance.
