2.2 Explanation

The first step is to find the performance bottlenecks of the current distributed JML prototype. This task involved following activities:

  • Choose the appropriate tool for performing the benchmark.
  • Use existing test data to measure the average proving process time of each theorem prover

3. Evaluation

3.1 Benchmarking Tool

There exists several free java profilers on the Internet. Following is a list of few java profilers that we found:

  • Eclipse TPTP: Eclipse built-in profiler. It provides framework and services for testing and collecting application performance measurement. Eclipse TPTP Website
  • DJProf: is an open source experimental java profiler. It adopts AspectJ mechanism to insert measurement points. DJProf Website
  • JMP: a profiler for monitoring/tracing objects usage and methods timing. It uses JVMPI (Profiler Interface) to collect measurements. JMP Website
  • Infrared: was developed for profiling J2EE application. Similar to DJProf, this profiler uses Aspect-Oriented Programming to insert performance monitoring code into the application. It provides more features than DJProf. Infrared Website

3.2 Distributed Technology

This section provides an overall description and the characteristics of three common distributed technologies: Java RMI, CORBA, and Web Service

3.2.1 Java RMI

  • Distributed Object Model
  • Communication usually blocked by firewalls, hence more appropriate in LAN environment.
  • Use TCP/IP protocol
  • Data exchanges in binary format
  • Limited to Java platform

3.2.2 CORBA

  • Multi-language RMI system
  • Communication is done through GIOP (General InterOrb?) or IIOP (Internet InterOrb?) protocol.
  • OS independent

3.2.3 Web Services

  • XML Based Message
  • Promotes SOA concept
  • Message driven Model
  • Interoperability between different platforms
  • Use SOAP protocol over HTTP/HTTPS
  • Data exchanges in native format (XML)
  • Low performance compared to Java RMI/CORBA

3.2.4 Hybrid

One of the possible solutions is to integrate different distributed technologies into the system. For example, integrate Java RMI and Web Services together. The goal is to combine the strength of each technology to solve our problem.

3.3 Caching

One of the approaches for optimizing performance is the use of caching mechanism.

3.3.1 Build vs. Buy

The principal question is whether we are going to develop the caching mechanism ourselves or using the existing one out of market.

  • OSCache: high performance J2EE caching framework. It provides caching of arbitrary objects, and support persistence caching. OSCache Website
  • Cache4j: a cache for Java objects. It provides in-memory caching, runs in multi-threaded environment, and allows user to choose a strategy among various caching algorithms. Cache4j Website
  • SwarmCache: simple lightweight distributed cache. It uses IP multicast to communicate within LAN environment. It is more appropriate for database driven application. SwarmCache Website
  • Java Caching System: distributed caching system for server-side applications. JCS Website
