JML4 Commit Notes - 2008-12-16

WHAT is in commit 1173?

Essentially added parsing (no resolution) support for method specification forall and old variable declarations.

DETAILS

Selected details:

  • Scanner updated to recognize "forall" and "old".
  • java.g: added appropriate production rules. Note that in defining the spec var declarations, use is made of the JDT LocalVariableDeclarationStatement production rule (seemed much better than duplicating all that code). This required adding a JmlParserHelper boolean field processingSpecVarDecls which is true precisely when spec var decls are expected. This allows us to have the method consumeEnterVariable() construction local declarations rather than fields.
  • JmlSpecCaseBody: added two fields of type JmlLocalDeclaration[] to hold forall and old spec var decls.
  • JmlConstants.ENABLE_SPEC_MERGE constant added to allow spec merging to be disabled programatically.
  • DbcTestCompiler: added test_0042_spec_variables() and test_0043_result_as_array(). The latter currently fails, though I will be working on that next.
