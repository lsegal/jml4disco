Use Case View

Developer Use Cases

Use Case 1: Enable Caching

Properties:

Primary Actor: Developer
Level: User-goal
Success Guarantee: Caching is enabled
Minimal Guarantee: ---

Preconditions:

  1. System is accessible
  2. Caching is disabled

Main Success Scenario:

  1. Primary Actor indicates to System that they wish to activate caching
  2. System provides a form for the Primary Actor to set the amount of disk space and/or memory to be dedicated to caching
  3. Primary Actor enters the appropriate data and submits
  4. System confirms that caching has been enabled

Extensions:

3.a. The data provided by the Primary Actor is wrong

3.a.1. The system returns an error message to the Primary Actor specifying why the data is not acceptable (if applicable)
3.a.2. Use case resumes at Step 3

Use Case 12: Remove Server

Properties:

Primary Actor: Admin
Level: User-goal
Success Guarantee: The system will no longer be able to utilize resources on remote machines specified.
Minimal Guarantee: ---

Preconditions:

  1. System is accessible
  2. Primary actor currently has access to a user interface for system configuration.

2.a.1.b.1 The list of remote machines from which the system can utilize resources is unchanged.

Use Case 13: Install/Setup Boogie

Properties:

Primary Actor: Admin
Level: User-goal
Success Guarantee: The system will now be able to use Boogie for verification.
Minimal Guarantee: ---

Preconditions:

  1. System is accessible
  2. Boogie is installed on the system.

Main Success Scenario:

  1. Primary actor specifies that he/she wants to configure settings for JML4 Disco.
  2. Primary actor specifies that he/she wants to install/setup Boogie.
  3. System prompts user for the path to Boogie.
  4. Primary actor specifies the path to Boogie.
  5. System acknowledges the path and presents the system as being setup to use Boogie.

Extensions:

5.a. Path Specified is invalid.

5.a.1. The system returns a warning indicating that the path specified by the user is invalid.

5.a.1.a The user corrects the specified path to Boogie.

5.a.1.a.1 The system goes to step 5.

5.a.1.b The user cancels the setup.

5.a.1.b.1 System does not setup Boogie.

Use Case 14: Enable Caching

Properties:

Primary Actor: Admin
Level: User-goal
Success Guarantee: Caching is enabled
Minimal Guarantee: ---

Preconditions:

  1. System is accessible
  2. Caching is disabled

Main Success Scenario:

  1. Primary Actor indicates to System that they wish to activate caching
  2. System provides a list of all currently "available" caching servers
  3. Primary Actor selects from the list and/or manually enters the desired servers and submits
  4. System provides a form for the Primary Actor to set the amount of disk space and/or memory to be dedicated to caching for each "available" caching server
  5. Primary Actor enters the appropriate data and submits
  6. System confirms that caching has been enabled for each server

Extensions:

2.a. There are no currently "available" caching servers

2.a.1. The system indicates there are no "available" servers and provides the manual entry form
2.a.2. Use case resumes at Step 3

5.a. The data provided by the Primary Actor is wrong

5.a.1. The system returns an error message to the Primary Actor specifying why the data is not acceptable (if applicable)
5.a.2. Use case resumes at Step 5
