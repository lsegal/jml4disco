Project Proposal

Team #2

Project: New static verification strategy for JML4
Client: Dependable Software Research Group(DSRG) - This group includes: Patrice Chalin, Perry James, George Karabotsos.

We'd be working with the DSRG to expand the JML4 project to include static verification of JML annotated Java programs using "Boogie". Boogie being a static program verifier for object oriented programs developped by microsoft. This project would entail translating the Abstract Syntax Tree (AST) into BoogiePL. The BoogiePL would in turn be passed to Boogie for further parsing and verification. Once Boogie is integrated as a proving strategy within JML4, stratic cheking times will be reduced and the process optimized through distributed and parallel verification.

Furthermore here are some other avenues which can be explored as part of a JML4 expansion, as mentioned by the client

  • There is a tool called JmlUnit that could be ported to the JML4 framework as well. This tool automatically generates a test oracle from the JML specs of a Java class.
  • Adapt Eclipse's refactoring to support the refactoring of specifications.
  • Automated generation of code from specifications.
  • Model Driven Design + JML --> code.
  • Use specs to optimize the compiler code.
